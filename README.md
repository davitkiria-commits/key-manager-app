# Key Manager App (PySide6)

Приложение для учета ключей по квартирам, выдачам и получателям.

## Версия

Текущая версия хранится в одном месте — в файле `version.txt`.
Это же значение используется в процессе сборки и в будущем может использоваться для автоматической проверки обновлений.

## Что умеет приложение

- Учет квартир и общего/выданного/утерянного количества ключей.
- Выдача и возврат ключей.
- Справочник получателей (сотрудников): добавление, редактирование, активация/деактивация.
- История операций с фильтрами по датам, человеку и квартире.
- Окно «Ключи на руках» (только незакрытые выдачи).
- Автосохранение всех данных в `data.json`.

## Простой запуск через `run.bat`

Основной способ запуска для разработки и ежедневного использования — через Python:

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Запустите:
   ```bat
   run.bat
   ```

`run.bat` запускает приложение командой `python main.py` (или `py -3 main.py`, если доступен launcher `py`).
Если Python или зависимости не установлены, скрипт покажет понятное сообщение и подскажет, что выполнить.

## Сборка portable EXE (папка `dist\key-manager`)

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Запустите сборку:
   ```bat
   build_exe.bat
   ```
3. Результат сборки (не `onefile`) находится в папке:
   ```
   dist\key-manager\
   ```
4. Основной исполняемый файл:
   ```
   dist\key-manager\key-manager.exe
   ```

## Как обновлять программу без потери данных

Приложение хранит данные в файле `data.json` рядом с `main.py` (для запуска через Python) или рядом с `key-manager.exe` (для portable-сборки).

### Обновление запуска через Python

1. Закройте приложение.
2. Обновите файлы проекта (например, через git pull / замену файлов).
3. Не удаляйте `data.json`.
4. Снова запустите `run.bat`.

### Обновление portable-сборки

1. Закройте `key-manager.exe`.
2. Возьмите новую папку `dist\key-manager` из свежей сборки.
3. Скопируйте в рабочую папку программы **все файлы, кроме `data.json`**, с заменой.
4. Убедитесь, что ваш старый `data.json` остался на месте.
5. Запустите `key-manager.exe`.

Так можно обновлять программу простой заменой файлов без полной пересборки на целевой машине.

## Подготовка к автообновлению

Базовая структура уже готова:

- `version.txt` — единый источник версии.
- `build_exe.bat` читает версию из `version.txt`.
- portable-формат (`dist\key-manager`) позволяет в будущем добавлять проверку «локальная версия vs удалённая версия» и частичную замену файлов.

## Данные

- Все данные автоматически сохраняются в `data.json` после каждого изменения.
- При запуске приложение загружает данные из `data.json`, если файл существует.
- Если файл отсутствует, приложение стартует с пустой базой.
- Если JSON повреждён, приложение покажет понятное сообщение и продолжит работу с пустыми данными.

### Какие данные сохраняются

- `apartments` — квартиры.
- `persons` — справочник людей (`id`, `full_name`, `role`, `is_active`).
- `active_issues` — незакрытые выдачи.
- `history` — история операций в расширенном формате.

## Краткий сценарий работы

1. Добавьте квартиры.
2. Откройте окно **«Получатели»** и добавьте людей.
3. В окне **«Выдать ключи»** выберите квартиру и получателя из выпадающего списка с поиском.
4. Используйте **«История»** и **«Ключи на руках»** для контроля состояния.
